Performance with iozone

Iozone is a public domain program executable on many unix platforms. Source may be downloaded and compiled for an executable capable of I/O analysis and production of I/O results in the form of a .wks file. The .wks file can be imported into Microsoft Excel and an 3-d chart produced to render I/O performance in the form of a graph.

The tests shown below were conducted with a V280 platform to a treefrog Sophomore array. The array consisted of three trays. A CSM attached to two expansion trays. 15K rpm drives in expansion tray #1 were used on a RAID0 and RAID5 volumes.

iozone source can be obtained from http://www.iozone.org

The gcc compiler was used to make the iozone executable with this make file. The makefile downloaded with the source distribution has a few compiler options found incompatible with the gcc used to compile and were removed.

This iozone binary was used for the test runs on Solaris 9.

iozone may be run in automatic mode. Automatic runs a number of tests. One can chart a test as shown below:

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

The chart may be selected and will be bordered by 4 sentinel squares on the edges of the chart image. The chart may now be copied and pasted into another document or application.

 

Two charts are shown here. These are from the WRITE sections of the raw files downloadable from the links below. The files contains other interesting and useful tests obtained from the Automatic mode of iozone.

Typically, iozone is run as:

./iozone -Rab <someResultsFile.wks> -g xG

where x = max file size desired for testing.

./iozone -h produces help.
