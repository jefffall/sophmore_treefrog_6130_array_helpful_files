This page will cover how to bring up a CSM out of the box. This experience would be similar to that of an unexperienced field engineer tasked with installing the CSM onto a live customer's system.  Typically, the Unibrick will arrive to a customer site and a field engineer will unpack it and move it to a location specified by the customer.  Now what?

Here is where real life (but in a lab), experience is applied to bring up the CSM enough allow the customer to see storage on-line.

What steps will we follow?

Here they are:
 

  • Unpack CSM - prepare CSM for initial power up. May involve removing special bolts and fixtures needed to hold internals in a rigid shippable state to avoid shipment damage.
  • Power UP CSM - check initial working status
  • Check firmware installed on CSM to make sure it is the correct version matching the desired Solaris MPxIO driver stack.
  • Check firmware installed on customer's HBA cards on server. Make sure the firmware is at acceptable levels and upgrade if needed.
  • Check base level of customer's Solaris installation to make sure basic acceptable patch levels are installed as per Sunsolve.
  • Check SAN driver stack to make sure MPxIO is installed and enabled
  • Check Fibre Channel HBA's for health and possible attached Fibre Channel Storage devices
  • Check FC switch to make sure domains and zoning are proper for this site.
  • Check if FC switch can see WWN's on Unibrick/CSM
  • Check if customer server (Solaris) can see WWN's from CSM through the switch
  • Create storage LUN(s) on the CSM/brick using TBD software or web interface.
  • Create proper /dev entries for desired storage
  • Format the storage for desired volume using newfs or similar tool to make a filesystem.
  • Mount the storage and test viability.
  • Test the Mulitplexing MpxIO by introducing a fault and observing error logs to make sure the injected I/O was handled properly.
  • Install and configure high level storage management software (capable of performing most of these tasks automatically).
