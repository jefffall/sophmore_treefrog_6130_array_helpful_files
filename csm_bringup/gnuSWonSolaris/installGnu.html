Installing GNU software onto NWS/VSS machines from the internet

There may be times whereas one needs to get a build environment set up on a machine pretty fast. Here is a method for downloading and installing GNU software for Solaris over the internet using a simple command. You'll get gcc installed in 10 minutes or less. You'll even get a 'hello world' program  working too. This rivals many of the Linux distro's that sport internet upgrades and installation.
What is cool is this is one stop shopping.  Locate what you want, command it to install and it does and over the internet from outside using the web proxy.

Say you want the latest gcc compliler on your lab machine or the machine on your desk. No problem. Install the software using the pkg-get command. This command is a small program relying on wget. Wget is a utility that speaks http and can get web pages thru firewalls. As we have a proxy here at Sun, we need to tell wget how to use the proxy via a wgetrc configuration file. Lastly, we may need to patch the intended target Solaris hosting system for any of this to work at all as it is patch level dependent.

Here are the proper steps: (must be root). Put all these (below) into a directory. You may need to run 'unzip' to decompress any zipped files. For the first three in the list below right click on them and save target.

  1. Download pkg-get
  2. Download wget
  3. Download wgetrc
  4. Download Solaris patch(s) (fixes auto installation problems)

Install pkg-get using: pkgadd

Install wget using: pkgadd

Copy wgetrc (config file) to /usr/local/etc. You will end up with the file called wgetrc in /usr/local/etc. Make sure it is readable.

It may be a good idea to install the patch too. It is actually two patches. To install the patch, unzip it. It will make a directory. cd into the directory and use 'pkgadd -d .'. It will pick up both patches. Tell it 'all' to install both of the patches it finds there.

Once all this is done, try to see what software you can install off of the internet. Do a cd to /opt/csw/bin

Try pkg-get as this:

./pkg-get
 (to see the help lines)

Try to see if you can connect to the archive and see what you can install:

pkg-get -a

At this point, pkg-get may complain it needs additional sofware. You will get a teaser about md5 missing. You can use pkg-add to install this missing checksum software. Then, gcc can be installed.

If you feel brave, install gcc2 (actually gcc)

./pkg-get -i gcc2    (this will install gcc for you)
