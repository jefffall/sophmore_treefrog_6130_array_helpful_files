MPXio Driver Stack - SAN Foundation Kit (SFK)

To leverage the greatest advantage from the CSM, it is useful to utilize multipathing. This way, if one path has a fault, the other can continue and data integrity is preserved.

This section of the Bringup page will attempt to allow a person to verify MPXio is installed and working properly on the Customer or lab machine.

Currently, the SAN Foundation Kit is experimental and under development.

One can get the latest SFK here: http://katana.central:8080/Binaries/SFS_Software/Beta/s9/sfk_6.5

Check for new SFK betas by browsing the directory here: http://katana.central:8080/Binaries/SFS_Software/Beta/s9/

Before installilng, it may be useful to refresh on some of the conceptual foundational components installed by the SAN Foundation kit. These include the Leadville driver stack inplemented by the  Leadville architecture and finally the MPXio features.


Installing the SFK:


Installation of the SFK version 6.5.2 can be tricky.

Once the tar file is obtained, untar it to a directory you have created. Be aware of a certain patch called Q959651-02. On most systems, running the installation script will return a code 24 whilst choking on patch Q959651-02. One can use the patchadd program to try to install the patch Q959651-02 manually. Invoking patchadd manually for patch Q959651-02 will show it depends on another patch already installed. Trying to remove that patch tells of another needing backing out before. There are dependencies at least three levels deep.

Typically, the "install_it" script is run and if it runs to completion successfully, then the SFK stack is assumed to be successfully installed. Note - at the end of the install, the install_it script attempts to run patch_pro. Patch pro will install correctly, but will then identify patches to install and fail at installing the suggested patches. This may be due to the experimental nature of the SFK. In any event, when the install_it script offers to install and run patch_pro, say no and consider the installation successful.

To see a sample install with a lot of problems with patch Q959651-02 and eventually installing successfully, please see sfk_6_5_2 install session log.
