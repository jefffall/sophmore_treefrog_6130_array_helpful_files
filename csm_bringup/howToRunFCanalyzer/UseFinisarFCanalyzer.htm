Using a Finisar Fibre Channel Analyzer

 

If the Finisar analyzer is broken or caught a virus:

Finisar builds fibre channel analyzers on PC platform using the NT 4.0 and Windows 2000 operating systems. Care must be taken to assure each analyzer has the proper firewall and anti virus software installed. This will insure trouble free operation of the PC based analyzer.

Finisar packages the analyzers on the Dolch Rugged PC platform. Various generations of this ruggedized portable PC exist. Some can only run NT 4.0 due to Trident video driver releases on the 9520 video chip. The later platforms use the Trident 9525DVD video chip. A Toshiba laptop video driver can be used to convert these systems to the more stable Windows 2000 professional or Server platform.

The Finisar software suite covers most OS platforms and most Finisar Fibre Channel test card releases.

The Finisar test suite can be downloaded from Finisar at:

ftp.finisar.com

The internal Sun firewall may prevent ftp access to this site.

http://www.dolch.com

The username for the support site is DolchRugged

the password is: DropMe

The username and password is case sensitive.

Dolch does not provide much Win2000 support. Drivers are typically for the Trident 9525DVD chip and the VIA Rhine II ethernet chip. Most any driver from many OEM vendors can be downloaded for the Finisar/Dolch in a pinch by performing a google search as Win2000 drivers are not on the Dolch or Finisar support sites.

Using the Fibre Channel Analyzer remotely.

Once the analyzer is connected between the initiator and target(s), it is useful to take a trace. Using the front panel of the analyzer can be cumbersome, so remote access is desired.

One can access a PC based analyzer using VNC Viewer, as long as VNC server is running on the Finisar Analyzer. VNC can be used from a Solaris based machine such as a Sunray or a standalone system or server.

Taking a Fibre Channel Trace

 

Most of the common Finisar tools are installed on the Desktop.

To take a trace, use the GT Trace controller.

 

Firstly, connect the analyzer to the initiator and target systems. this is done by pulling the cables out of the switch or HBA, running the cables to the analyzer and then plugging the cables first pulled out of the HBA or switch into the analyzer.

The analyzer will have one or more cards called "Channels". Each has two ports. One uses both ports on the card - one for in from the HBA or switch  (initiator) and one out to the target array, switch or device.

Now, run GT Trace controller:

Make sure it is listed as ready.

Take a quick look with VIEW:

One can see "loss of sync".

This is normal at the FC-0 layer to have an error when there is no cable connected.

So, if these errors are encountered, the unit is probably not connected.
