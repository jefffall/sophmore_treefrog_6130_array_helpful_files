Windows Server 2003

2nd static IP on an ethernet adapter

 

Assure the server has a static IP, not DHCP IP

 

Add another IP by selecting Advanced...

Selct ADD

Add in the proper IP to be on the same subnet as the array controllers A and B.

Controller A will be set to 192.168.128.101

Controller B will be set to 192.168.128.102

192.168.128.100 is used for the second IP address in this example as it does not conflict with above, yet is on the same subnet - eg first three octets match:

This finishes the configuration:

As the array is on the subnet, a new gateway is not needed.

Use 6130 Configuration to add the array:

 

Select Auto Discovery. If that does not work, use manual discovery and enter one of the static default  IP addresses of one of the controllers, then add the other.

For example:

Use 192.168.128.101

For discovery.

Then, use 192.168.128.102

When this is complete, use SE6130 Configuration Service to assign a new IP to each controller.

Make sure you can then contact the controllers on the normal subnet with ping and using SE6130 Configuration Service.

Then, in Windows 2003 server delete the extra 102.168.129.100 IP address under Network Connections/Advanced.

Remove:

Done.
